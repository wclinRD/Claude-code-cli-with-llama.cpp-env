# Reflection Skill (反射技能)

## 功能說明

當任何技能工具使用失敗後，此技能會自動進行以下工作：

1. **分析失敗原因**：識別錯誤類型和根本原因
2. **學習成功方法**：記錄什麼做法能導致成功
3. **更新知識庫**：將學習結果儲存到技能檔案中
4. **提供改進建議**：給出具體的避免錯誤指導

## 使用方式

### 自動觸發
- 當任何技能工具調用失敗後自動觸發
- 系統會自動記錄錯誤訊息和成功解決方案

### 手動觸發
```bash
/reflection --skill-name "<skill 名稱>" --context "<失敗情境>"
```

## 錯誤分析類型

### 1. 工具失敗
- 工具名稱不正確
- 參數格式錯誤
- 缺少必要參數
- 工具不在可用列表中

### 2. 內容失敗
- 分析結果不準確
- 建議不切實際
- 未解決用戶問題

### 3. 格式失敗
- 輸出格式錯誤
- 數據格式不正確

## 錯誤處理流程

```
1. 檢測失敗
2. 分析錯誤類型
3. 記錄失敗情境
4. 學習成功方法
5. 更新知識庫
6. 提供改進建議
```

## 更新規則

### 成功學習規則
- 記錄失敗原因
- 記錄成功方法
- 提供具體避免錯誤的指導

### 知識庫更新格式
```
[日期] 錯誤類型：<類型>
問題：<問題描述>
原因：<根本原因分析>
成功方法：<具體做法>
改進建議：<如何避免類似錯誤>
```

## 示例

### 工具使用失敗
**錯誤情境**：使用 `/github-repo` 時工具名稱錯誤

**失敗原因**：
- 工具名稱寫錯了
- 未檢查可用工具列表

**成功方法**：
- 使用正確的工具名稱
- 先使用 `/skills` 查看可用工具

**改進建議**：
- 使用前使用 `/skills` 查看可用工具
- 仔細檢查工具名稱拼寫

### 分析建議失敗
**錯誤情境**：使用 `/stock-analysis` 時未提供足夠資訊

**失敗原因**：
- 未指定股票代號
- 未說明分析需求

**成功方法**：
- 提供完整的分析需求
- 指定具體的股票或 ETF

**改進建議**：
- 使用完整描述
- 提供具體的股票代號
- 說明分析重點

## 注意事項

1. 每次失敗後都會自動記錄
2. 成功方法會被優先記錄
3. 知識庫會隨著使用不斷更新
4. 改進建議會具體化
